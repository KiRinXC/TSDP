## 关键结论

BN gamma 是最稳定且成本最低的遗漏语义。它只增加 `4,800` 个参数，即模型参数的 `0.0428%`，却在全部八个 k 上同时降低 accuracy 和 fidelity、提高 KL。Top-10 加入 BN gamma 后，accuracy/fidelity 从 `0.2566/0.2783` 降至 `0.1928/0.2104`；Top-11 加入后达到 `0.1748/0.1862`，已经优于原始 Top-17 的 `0.1804/0.1959`，但保护比例只有 `14.2894%`，而原始 Top-17 为 `98.2985%`。

按三项原始指标直接与当前 soft 黑盒比较，以下四点同时达到或越过参考线：

```text
组合                参数比例   accuracy  fidelity  posterior KL
Top-16 + BN gamma    93.0880%     0.1451    0.1528       2.871881
Top-17 + BN gamma    98.3413%     0.1319    0.1436       2.915245
Top-16 + BN gamma
       + Downsample  94.6202%     0.1470    0.1518       2.878648
Top-17 + BN gamma
       + Downsample  99.8735%     0.1348    0.1420       2.943139
soft full protection 100.0000%    0.1545    0.1610       2.835290
```

这验证了 BN gamma 能补齐全部 eligible 主路径 weight 的剩余攻击信息，但当前达到黑盒的保护成本仍然过高，不能作为最终低成本方案。后续通道块方法应把对应 BN gamma 作为通道节点状态的一部分，并在较小的卷积块集合中寻找同样效果。

Downsample Conv 在全部 k 上也产生方向一致的保护增益，固定增加 `172,032` 个参数，即 `1.5322%`。其中 Top-13 + Downsample Conv 达到 `0.1604/0.1700/2.739298`，是较低 k 下最接近黑盒的点；Top-17 + Downsample Conv 的 accuracy 已低于黑盒，但 fidelity 高 `0.0007`、KL 低 `0.029651`，没有在三项指标上同时达到参考线。Downsample Conv 应保留在后续计算图候选中，但优先级低于 BN gamma。

BN gamma 与 Downsample Conv 的组合固定增加 `176,832` 个参数，即 `1.5749%`。Top-10 至 Top-12 时，组合在三项指标上均优于两个单项，说明二者在较小前缀下存在互补；但 Top-13 至 Top-15 不再稳定叠加，Top-15 甚至同时弱于两个单项。Top-16/17 组合仍达到黑盒，且 fidelity/KL 略优于 BN gamma 单项，但 accuracy 分别高 `0.0019/0.0029`，保护成本又多 `1.5322%`。因此该组合没有把首次达到黑盒的 k 从 16 提前，也没有支配 BN gamma 单项；它支持 downsample 作为后续计算图候选，却不支持无条件把全部 downsample 与全部 BN gamma 绑定保护。

Stem Conv 的独立作用很弱。它对 KL 的增量稳定在约 `0.0344–0.0515`，但 accuracy/fidelity 改善很小；Top-12 accuracy 和 Top-14 fidelity 还出现轻微反向变化。当前没有证据支持把 Stem Conv 作为主要攻击瓶颈，但也不能据此否定更细通道级 Stem 节点与其他路径的组合贡献。

三类额外语义的并集没有复现单项贡献的简单叠加。多数 k 上，`all_extras` 明显弱于只加入 BN gamma 或 downsample Conv；Top-14 的 accuracy 甚至比原始 Top-14 高 `0.0019`。在当前固定攻击器中，保护更多状态会改变公开/victim 初始化组合及后续优化轨迹，因此经验 MS 指标不保证严格单调。该现象说明这些结果只能解释为当前攻击协议下的经验安全效果，不能直接视为信息论意义上的保护单调性；后续还需考虑攻击者主动忽略部分已暴露状态的更强初始化选择。

Top-17 + All extras 保护 `99.9572%` 参数，结果为 `0.1572/0.1642/2.790213`。它与独立 Lab05 `weight` 组的 `0.1582/0.1670/2.785335` 分别只差 `-0.0010/-0.0028/+0.004878`，符合“全部 weight 再增加 100 个分类头 bias”的集合关系，说明两套语义实现和实验轨迹相互一致。
