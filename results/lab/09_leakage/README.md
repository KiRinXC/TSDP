# 实验 09 结果

本实验固定 Lab06 的 5.7529% 系统保护集合，只改变攻击者对同一批已泄露 victim
状态的利用强度。0% 是完全忽略泄露状态的 matched soft 黑盒初始化，100% 是当前
直接复制全部未保护状态的混合初始化。25%、50% 和 75% 均由攻击者使用已知 public
与 victim 状态主动构造，不增加泄露信息、query 或可训练参数。

## 实验结论

100% 直接复制确实构造了严重不自洽的 public/victim 混合初始化。其训练前
query-validation soft CE 从黑盒的 4.76294 升至 11.47155，说明负迁移在读取任何
query 梯度之前已经出现；经过全参数 finetune 后，100% 端点仍在 10/10 seed 上
表现为 accuracy 和 fidelity 更低、posterior KL 更高。

但这不能支持“未保护 tensor 让攻击者真正失明”。25%、50% 和 75% 在 10/10 seed
上都同时优于黑盒，说明未保护状态包含稳定可利用的 victim 信息。当前标准攻击失败
的原因是完整照搬状态差异过度破坏初始化兼容性，而不是泄露状态没有信息。

50% 是当前五点中稳定最强的利用强度，相对黑盒平均变化为：

```text
MS accuracy    +0.04799
Fidelity       +0.05285
Posterior KL   -0.42334
Validation CE  -0.53720
```

只使用各 seed 的 100 条 query-validation 选择利用强度时，适应性攻击者在 10/10
seed 上全部选择 50%，不需要读取 `eval_ms`。其最终均值就是
`0.19574/0.20706/2.56165`。因此 100% 混合初始化低于黑盒是一种可以被攻击者通过
状态收缩绕开的优化陷阱，不能作为当前保护集合达到经验黑盒等效的证据。

本实验回答了“负迁移是否真实存在”：答案是存在，而且在 100% 端点训练前就很强；
同时也回答了“攻击者能否利用泄露状态”：答案同样是能。它尚未回答为什么固定的
保护位置会使完整拼接发生如此剧烈的非线性失配，也没有证明 50% 是连续区间中的
全局最优强度。
