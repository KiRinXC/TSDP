# 实验 04 结果

本目录记录 TensorShield 作者 eligible rank 的 Top-1～Top-17 前缀、三个位置集合和
Top-12 删除消融。全部训练 case 均采用 400/100
soft-posterior validation-best 协议；完整 Top-12 等复用点只读取同协议来源，
不重复训练。

## 实验结论

作者 eligible rank 确实包含有效的位置排序信号，但不是由参数量单独解释：前十项只
保护 `14.2467%` 参数就比保护 `94.0303%` 参数的后十项更强。Top-12 以后前缀曲线
基本进入平台，扩到 Top-17、保护 `98.2985%` 参数仍未同时达到 soft 黑盒，说明固定
eligible weight 列表没有覆盖形成攻击能力的全部状态语义。

删除消融进一步表明 rank 成员具有强条件依赖。分类头 weight（rank-3）被删除后攻击
从 `0.1403/0.1519/2.856598` 明显反弹到 `0.2216/0.2464/2.317533`，是最清晰的必要
项；但删除 rank-5 或 rank-10 反而让固定攻击略弱，所以不能把 rank 前缀解释成每一项
独立、单调贡献。Lab04 支持“位置和组合关系重要”，不支持仅凭 Top-k 次序推出串行
因果路径；后续十种子候选验证已经迁到 Lab06。

前十项以远低于后十项的参数量取得更强保护，表明位置语义比单纯参数量更重要。

## Top-12 消融

完整 Top-12 为 `0.1403/0.1519/2.856598`。删除分类头 weight（rank-3）后反弹到
`0.2216/0.2464/2.317533`，是最明确的单项依赖。删除 rank-5 或 rank-10 反而使
固定攻击略弱；删除 rank-6、rank-7 或二者同时删除均使攻击反弹。以删除
rank-5/8/10 为基准的 2×2 因此不再支持“下游 unit 48 使上游 unit 36 条件冗余”
的串行解释。
