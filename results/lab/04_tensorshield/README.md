Top-12 删除消融和四策略十种子候选验证。全部训练 case 均采用 400/100
soft-posterior validation-best 协议；完整 Top-12 等复用点只读取同协议来源，
不重复训练。

## Top-12 消融

完整 Top-12 为 `0.1403/0.1519/2.856598`。删除分类头 weight（rank-3）后反弹到
`0.2216/0.2464/2.317533`，是最明确的单项依赖。删除 rank-5 或 rank-10 反而使
固定攻击略弱；删除 rank-6、rank-7 或二者同时删除均使攻击反弹。以删除
rank-5/8/10 为基准的 2×2 因此不再支持“下游 unit 48 使上游 unit 36 条件冗余”
的串行解释。

## 四种策略的十种子结果

产生 400/100 query 划分并重放 canonical 初始化；soft full-protection 也按相同
seed 和划分训练，仅作为参考线。四种策略都保护分类头 weight/bias。

配对结果给出三个观察。

第一，BN gamma 相对 Top-10 在 `10/10` seed 上都同时降低 accuracy、降低 fidelity
并提高 posterior KL。配对均值差 `(+BN)-Top10` 为
`-0.04141±0.00355 / -0.04644±0.00368 / +0.34974±0.04081`。因此 BN gamma
不是 seed-42 偶然现象，而是当前模型与攻击协议下稳定的遗漏保护语义。

第二，在已经保护 BN gamma 的条件下，暴露 rank-5/8/10 后仍在 `10/10` seed 上
同时得到更低 accuracy/fidelity 和更高 KL。配对均值差
`candidate-(Top10+BN)` 为
`-0.01161±0.00542 / -0.01239±0.00562 / +0.12009±0.04006`。这说明删点集合
没有在任何一个评估 seed 上引发当前固定攻击反弹，并把保护比例从 `9.0362%` 降到
`7.0662%`。由于暴露更多信息在信息意义上不可能增强保护，数值进一步越线应解释为
固定 surrogate 优化的负迁移，而不能写成 rank-5/8/10 的暴露提升了安全性。

相对逐 seed soft 黑盒，Top-10 为三项同时达到 `0/10`，Top-10+BN gamma 为
`8/10`，删除 5/8/10 为 `9/10`，额外删除 rank-6 后为 `10/10`。新消融相对原候选
的配对均值差为
`-0.00716±0.00639 / -0.00614±0.00620 / +0.03820±0.05463`；`8/10` seed
三项同时改善。seed 47 仅 KL 小幅反向 `-0.01014`，seed 51 三项均反向，但该 seed
仍达到 matched 黑盒边界。相对 `Top-10+BN gamma`，新消融在 `10/10` seed 三项
同时更强，均值差为
`-0.01877±0.00465 / -0.01853±0.00531 / +0.15828±0.05399`。

因此在当前固定攻击下，保护 rank-6 `layer2.0.conv2.weight` 没有带来稳定收益；
只保留 Top-10 中的五个 `conv1.weight`、分类头 weight/bias 和全部 BN gamma，
已用 5.7529% 参数在十个 seed 上达到经验黑盒边界。这与“阶段 1–3 的 conv1
比相邻 conv2 更值得保护”的观察一致，但还不能推广为所有模型、所有层或所有
conv2 的先验规则。暴露更多信息在信息意义上不可能增强保护，越过黑盒的数值仍应
解释为当前 surrogate 优化的负迁移，而不是额外暴露 rank-6 提高了固有安全性。

两组候选仍由 MS 消融反馈后验选出，只证明后验集合及 rank-6 消融在独立 seed 上的
经验稳定性，不是无偏的计算图或 XAI 先验选择方法。
